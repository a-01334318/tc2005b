Preguntas y Respuestas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras interconectadas, mientras que la World Wide Web (WWW o Web) es un conjunto de recursos de información en línea, como páginas web, imágenes, videos y otros archivos multimedia, que se acceden a través de Internet.

¿Cuáles son las partes de un URL?

Un URL (Uniform Resource Locator) está compuesto por varias partes, que incluyen el protocolo (http, https, ftp, etc.), el nombre de dominio (como google.com), la ruta del recurso (como /search), y posiblemente parámetros adicionales, como un identificador de sesión o una cadena de consulta.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los métodos HTTP se utilizan para indicar el tipo de acción que se debe realizar en un recurso web determinado. Algunos de los métodos HTTP comunes incluyen:

GET: recuperar una página o recurso web. HEAD: recuperar los encabezados de un recurso web sin descargar todo el contenido. POST: enviar datos a un servidor para procesarlos (como enviar un formulario). PUT: actualizar un recurso web existente. PATCH: actualizar parcialmente un recurso web existente. DELETE: eliminar un recurso web existente.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST al enviar un formulario HTML, ya que permite enviar datos al servidor para que sean procesados y almacenados en una base de datos. Además, el método POST oculta los datos enviados de forma más segura que el método GET, que coloca los datos en la URL de forma visible.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método HTTP utilizado para acceder a una página a través de un URL es GET. El navegador envía una solicitud GET al servidor para recuperar el recurso web (como una página HTML) y mostrarlo al usuario.

¿Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código de respuesta HTTP 200 indica que la solicitud fue exitosa y que el servidor devolvió el recurso solicitado. En otras palabras, no hubo errores en el proceso.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 404, que indica que el recurso solicitado (como una página web) no se encontró en el servidor. Esto puede deberse a un error en la dirección URL o a que el recurso se eliminó o movió sin actualizar los enlaces correspondientes. El desarrollador debe corregir el error y redirigir al usuario al recurso correcto.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 500. El error 500 indica un error interno del servidor, lo que significa que el problema no está en la computadora del usuario, sino en el servidor que aloja el sitio web. Es responsabilidad del desarrollador investigar y solucionar el problema en el servidor para asegurarse de que el sitio web esté disponible para todos los usuarios.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo HTML5 depreciado o desaprobado significa que la especificación HTML5 ya no recomienda su uso. Los atributos desaprobados se mantienen en HTML5 por razones de compatibilidad, pero los desarrolladores deben evitar usarlos. Algunos elementos de HTML 4 que están desaprobados en HTML5 son:

  • <acronym>
  • <applet>
  • <basefont>
  • <big>
  • <center>
  • <font>
  • <frame>

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las diferencias principales entre HTML4 y HTML5 son:

  • HTML5 incluye nuevas etiquetas para mejorar la semántica del código, como <atarticle>, <header>, <footer>, <section>, etc.
  • HTML5 admite contenido multimedia incorporado, como video y audio, sin necesidad de complementos de terceros.
  • HTML5 incluye formularios mejorados y nuevos elementos de formulario, como la validación de formulario HTML5, que es más fácil de usar y más potente que la validación de formulario HTML4.
  • HTML5 admite la creación de aplicaciones web sin conexión y proporciona almacenamiento en caché del lado del cliente.
  • HTML5 utiliza un modelo de procesamiento más flexible que HTML4, lo que hace que sea más fácil para los navegadores interpretar el código y mostrar las páginas web de manera consistente.

¿Qué componentes de estructura y estilo tiene una tabla?

Los componentes de estructura y estilo de una tabla HTML son:

  • <table> : la etiqueta de inicio y fin de la tabla.
  • <tr> : representa una fila de la tabla.
  • <th> : representa una celda de encabezado de la tabla.
  • <td> : representa una celda de datos de la tabla.
  • <caption> : proporciona un título para la tabla.
  • <thead>, <tfoot>, <tbody> : dividen la tabla en secciones de encabezado, pie y cuerpo respectivamente.
  • <colgroup>, <col> : especifican propiedades de estilo para columnas.

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma HTML5 son:

  • <input> : utilizado para crear campos de entrada de texto, casillas de verificación, botones de radio, botones de selección, etc.
  • <textarea> : utilizado para crear áreas de texto multilínea.
  • <select> : utilizado para crear menús desplegables y listas de selección.
  • <button> : utilizado para crear botones.
  • <datalist> : utilizado para crear una lista de opciones para un campo de entrada de texto.
  • <output> : utilizado para mostrar los resultados de una operación.

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información es un modelo que describe las diferentes etapas que un sistema de información atraviesa desde su concepción hasta su desaparición. El ciclo de vida típico de un sistema de información se compone de las siguientes fases:

  • Planificación: se definen los objetivos del sistema, se identifican los requisitos y se evalúa la viabilidad del proyecto.
  • Análisis: se investiga y analiza el problema a resolver, se documentan los procesos actuales, se identifican las necesidades de los usuarios y se definen los requisitos funcionales y no funcionales del sistema.
  • Diseño: se establecen las especificaciones técnicas, se diseñan los componentes del sistema y se define la arquitectura del mismo.
  • Desarrollo: se construye y se prueba el sistema según las especificaciones definidas en las fases anteriores.
  • Implementación: se realiza la instalación del sistema en el entorno de producción, se realiza la formación de los usuarios y se prepara la documentación del sistema.
  • Operación y mantenimiento: se lleva a cabo la explotación del sistema, se realizan ajustes y mejoras según las necesidades y se proporciona soporte a los usuarios.
  • Desaparición: el sistema se desactiva, se retira del entorno de producción y se archiva toda la documentación relevante.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información es una metodología que describe cómo desarrollar un sistema de información en cada una de las etapas del ciclo de vida. El ciclo de desarrollo típico de un sistema de información se compone de las siguientes fases:

  • Análisis de requisitos: se identifican y documentan los requisitos del sistema.
  • Diseño del sistema: se diseña la arquitectura del sistema y se especifican los componentes.
  • Implementación: se construyen los componentes y se integran en el sistema.
  • Pruebas: se realizan pruebas para verificar que el sistema cumpla con los requisitos.
  • Implementación y mantenimiento: se realiza la implementación del sistema en el entorno de producción y se lleva a cabo el mantenimiento del sistema durante su ciclo de vida.
  • Retirada: se realiza la retirada del sistema una vez que ya no es necesario.

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Como ingeniero de software, recomendaría evitar el uso excesivo de !important en CSS, ya que puede dificultar la lectura del código y dificultar la realización de cambios futuros.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Al seleccionar una imagen de fondo en una página HTML, es importante considerar la calidad y el tamaño del archivo de la imagen. Las imágenes grandes y pesadas pueden aumentar significativamente el tiempo de carga de la página, lo que puede afectar negativamente la experiencia del usuario. Se debe elegir una imagen de alta calidad que sea lo suficientemente pequeña para no afectar el rendimiento de la página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Al elegir las unidades de un estilo, es importante considerar la naturaleza del elemento que se está estilizando y el contexto en el que se utiliza. Por ejemplo, las unidades de porcentaje (%), pueden ser útiles para establecer dimensiones y márgenes relativos al tamaño del contenedor. Las unidades de píxeles (px) son útiles para establecer dimensiones y márgenes absolutos, y las unidades de puntos (pt) se utilizan comúnmente para estilos de tipografía en dispositivos de alta resolución.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Al eliminar espacios en blanco, comentarios y otros caracteres innecesarios del archivo CSS, se puede reducir su tamaño en gran medida, lo que mejora el rendimiento del sitio web.
